Detailed Description

Represents an observable 3D object.

Contains a vertex_data, a set of uniform collections, a shader, animations, a skeleton.

Warning
I think this class contains a bit too much implementation. (see draw method). It seems weird that "model" is responsible for binding and clearing all uniform data for the shader.

the shaderprogram and uniform data (textures etc.) Should probably be broken out into a new abstraction. This work would be a good match for the "material" class seen in many engines.
